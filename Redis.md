# 🔴 Redis (Remote Dictionary Server)

> **Redis** — это высокопроизводительная NoSQL база данных класса **In-memory**. Чаще всего используется как сверхбыстрое хранилище данных, кэш или брокер сообщений.

---

## ⚡️ Ключевые особенности

*   **Скорость:** Благодаря работе напрямую с RAM, Redis способен обрабатывать **100,000+ запросов в секунду** с минимальными задержками.
*   **Гибкость данных:** В отличие от Memcached, поддерживает не только простые строки, но и сложные структуры:
    *   `Strings`, `Lists`, `Sets`, `Hashes`, `Sorted Sets (ZSets)`.
*   **Атомарность:** Все операции выполняются в **один поток**. Это гарантирует, что две команды не изменят один и тот же объект одновременно (защита от *Race Condition*).
*   **Persistence:** Поддерживает сохранение данных на диск (RDB/AOF), что делает её полноценной базой, а не просто временным кэшем.

## 🛠 Базовые команды (Cheat Sheet)

### 🔑 Управление ключами (Keys)


| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `SET` | Создать или обновить ключ | `SET user:1 "Alice"` |
| `GET` | Получить значение ключа | `GET user:1` |
| `DEL` | Удалить ключ и его значение | `DEL user:1` |
| `EXISTS` | Проверить наличие (1 - есть, 0 - нет) | `EXISTS user:1` |
| `EXPIRE` | Установить время жизни в секундах | `EXPIRE user:1 60` |
| `TTL` | Узнать, сколько секунд осталось жить | `TTL user:1` |

---

### 📝 Работа со строками (Strings)
```bash
INCR counter          # Увеличить число на 1
DECR counter          # Уменьшить число на 1
APPEND key " suffix"  # Дописать строку в конец существующей

### 📋 Списки (Lists)
*Списки в Redis — это связные списки строк. Идеально подходят для очередей (Queue).*


| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `LPUSH` | Добавить элемент в **начало** списка | `LPUSH tasks "buy_milk"` |
| `RPUSH` | Добавить элемент в **конец** списка | `RPUSH tasks "clean_room"` |
| `LPOP` | Извлечь и удалить **первый** элемент | `LPOP tasks` |
| `RPOP` | Извлечь и удалить **последний** элемент | `RPOP tasks` |
| `LRANGE` | Получить диапазон элементов | `LRANGE tasks 0 -1` (все) |

---

### 🗄 Хеши (Hashes)
*Хеши — это объекты (карты). Лучший способ хранить профили пользователей или товары.*

```
HSET user:100 name "Ivan" age 25  # Создать объект с полями
HGET user:100 name                # Получить конкретное поле
HGETALL user:100                  # Получить весь объект целиком
HINCRBY user:100 age 1            # Увеличить числовое поле на N
```
