# 🔴 Redis (Remote Dictionary Server)

> **Redis** — это высокопроизводительная NoSQL база данных класса **In-memory**. Чаще всего используется как сверхбыстрое хранилище данных, кэш или брокер сообщений.

---

## ⚡️ Ключевые особенности

*   **Скорость:** Благодаря работе напрямую с RAM, Redis способен обрабатывать **100,000+ запросов в секунду** с минимальными задержками.
*   **Гибкость данных:** Поддерживает сложные структуры: `Strings`, `Lists`, `Sets`, `Hashes`, `Sorted Sets (ZSets)`.
*   **Атомарность:** Все операции выполняются в **один поток**, что гарантирует защиту от *Race Condition*.
*   **Persistence:** Поддерживает сохранение данных на диск (RDB/AOF), что делает её полноценной базой.

## 🛠 Базовые команды (Cheat Sheet)

### 🔑 Управление ключами (Keys)

*Эти команды работают с любым типом данных (Strings, Lists, Hashes и т.д.).*


| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `DEL` | Удалить ключ и его содержимое | `DEL user:1` |
| `EXISTS` | Проверить наличие ключа (1 - есть, 0 - нет) | `EXISTS user:1` |
| `TYPE` | Узнать тип данных (string, list, hash...) | `TYPE user:1` |
| `RENAME` | Переименовать ключ (перезапишет старый) | `RENAME old new` |
| `RENAMENX` | Переименовать, только если новое имя свободно | `RENAMENX k1 k2` |
| `COPY` | Копировать ключ в новое имя | `COPY k1 k2` |
| `RANDOMKEY` | Вернуть случайный ключ из базы | `RANDOMKEY` |

#### ⏱ Управление временем жизни (TTL)


| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `EXPIRE` | Установить время жизни в секундах | `EXPIRE key 60` |
| `PEXPIRE` | Установить время жизни в миллисекундах | `PEXPIRE key 1000` |
| `TTL` | Узнать, сколько секунд осталось жить | `TTL key` |
| `PTTL` | Узнать, сколько миллисекунд осталось | `PTTL key` |
| `PERSIST` | Удалить таймер (сделать ключ вечным) | `PERSIST key` |

#### 🔍 Поиск и перебор (Администрирование)


| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `KEYS` | Поиск по маске (медленно, опасно на prod) | `KEYS user:*` |
| `SCAN` | Итерация по ключам (безопасно для prod) | `SCAN 0 MATCH user:*` |
| `DBSIZE` | Показать общее количество ключей в базе | `DBSIZE` |
| `FLUSHDB` | Удалить **все** ключи в текущей базе | `FLUSHDB` |
| `FLUSHALL` | Удалить **все** ключи во всех базах данных | `FLUSHALL` |

### 📝 Работа со строками (Strings)

#### 基础 (Базовые операции)

| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `SET` | Установить значение ключа | `SET user:name "Alex"` |
| `GET` | Получить значение ключа | `GET user:name` |
| `GETSET` | Установить новое значение и вернуть старое | `GETSET status "online"` |
| `MSET` | Установить сразу несколько ключей | `MSET k1 "v1" k2 "v2"` |
| `MGET` | Получить значения нескольких ключей | `MGET k1 k2` |
| `STRLEN` | Возвращает длину строки | `STRLEN user:name` |
| `APPEND` | Дописывает строку в конец существующей | `APPEND msg "!!!"` |

#### ⏱ Управление временем (TTL)

| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `SETEX` | Установить значение и время жизни (сек) | `SETEX sess 60 "data"` |
| `PSETEX` | То же самое, но в миллисекундах | `PSETEX sess 1000 "d"` |
| `SETNX` | Установить, только если ключа **не существует** | `SETNX lock "true"` |

#### 🔢 Работа с числами (Атомарные счетчики)
*Если в строке число, Redis умеет с ним работать напрямую.*


| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `INCR` | Увеличить целое число на 1 | `INCR hits` |
| `DECR` | Уменьшить целое число на 1 | `DECR hits` |
| `INCRBY` | Увеличить на конкретное число | `INCRBY hits 10` |
| `DECRBY` | Уменьшить на конкретное число | `DECRBY hits 5` |
| `INCRBYFLOAT` | Увеличить на дробное число | `INCRBYFLOAT price 0.5` |

#### ✂️ Работа с подстроками (Range)

| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `GETRANGE` | Получить часть строки (с...по) | `GETRANGE key 0 3` |
| `SETRANGE` | Заменить часть строки с позиции | `SETRANGE key 5 "Redis"` |

#### 🏁 Работа с битами (Bitmaps)
*Strings в Redis можно использовать как массив битов (очень экономно).*


| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `SETBIT` | Установить бит (0 или 1) в позиции | `SETBIT visits 10 1` |
| `GETBIT` | Получить значение бита в позиции | `GETBIT visits 10` |
| `BITCOUNT` | Посчитать количество установленных бит (1) | `BITCOUNT visits` |
| `BITOP` | Побитовые операции (AND, OR, XOR, NOT) | `BITOP AND res k1 k2` |

### 📋 Работа со списками (Lists)
*Списки в Redis — это связные списки строк. Идеально для очередей (Queue) и стеков.*

| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `LPUSH` | Добавить в **начало** списка | `LPUSH tasks "task1"` |
| `RPUSH` | Добавить в **конец** списка | `RPUSH tasks "task2"` |
| `LPOP` | Извлечь и удалить **первый** элемент | `LPOP tasks` |
| `RPOP` | Извлечь и удалить **последний** элемент | `RPOP tasks` |
| `LLEN` | Получить длину списка | `LLEN tasks` |
| `LRANGE` | Получить элементы по индексам | `LRANGE tasks 0 -1` |
| `LREM` | Удалить N элементов, равных значению | `LREM tasks 1 "task1"` |
| `LTRIM` | Обрезать список (оставить диапазон) | `LTRIM tasks 0 99` |
| `LINDEX` | Получить элемент по индексу (без удаления) | `LINDEX tasks 1` |
| `LSET` | Изменить значение элемента по его индексу | `LSET tasks 0 "new_task"` |
| `LINSERT` | Вставить элемент ДО или ПОСЛЕ другого | `LINSERT tasks BEFORE "t2" "t1"` |
| `LPOS` | Найти индекс (позицию) элемента по значению | `LPOS tasks "task1"` |
| `RPOPLPUSH` | Перенести элемент из одного списка в другой | `RPOPLPUSH q1 q2` |
| `LMOVE` | Атомарно переместить элемент (аналог RPOPLPUSH) | `LMOVE q1 q2 LEFT RIGHT` |
| `LPUSHX` | Добавить в начало, только если список существует | `LPUSHX tasks "task0"` |
| `RPUSHX` | Добавить в конец, только если список существует | `RPUSHX tasks "task3"` |
| `BRPOP` | Блокирующее извлечение с конца (с ожиданием) | `BRPOP tasks 30` |
| `BLPOP` | Блокирующее извлечение из начала (с ожиданием) | `BLPOP tasks 30` |
| `BLMOVE` | Блокирующее перемещение элемента с тайм-аутом | `BLMOVE q1 q2 LEFT RIGHT 10` |
